IEI.os 10 X Current

IEI.os is the operating system of IEI Incorporated, continuously developed since 1993. Designed to run on IEI hardware and the Penter CPU family, IEI.os 10 X brings a modern desktop experience with deep kernel integration.

Features — IEI.os 10 X

LEXT Desktop

LuaExtension tiling window manager. Mac-inspired TUI with mouse support, a menu bar, dock, and draggable windows.

IASH v2.0

IEI Advanced Shell with cynext command loading, cooperative process scheduling and full VFS integration.

Cynessa Kernel

Microkernel architecture with cooperative coroutine scheduling, IPC message queues and syscall interface.

cynext Packages

JSON-defined shell extensions. Drop a .cynext file into /cynext/ and commands are available immediately on next boot.

Cynessa Microkernel

Cynessa is a pure Lua userspace microkernel that runs on any POSIX host. Cooperative coroutine scheduling, a virtual filesystem, IPC, and the cynext shell extension system — all in a package small enough to fit on a zip disk.

$ lua bootloader.lua

[OK] Kernel: cynessa.lua
[OK] Disk: cynessa.cyd
Handing control to Cynessa...
[Cynessa] Kernel starting...
[Cynessa] Mounting ramdisk: cynessa.cyd
[init] PID 1 starting
[init] starting shell
Cynessa Shell v2.0 - IEI Inc.
cynext command system active
csh> _

Microkernel

Process table, cooperative round-robin scheduler, IPC message queues and a clean syscall interface.

.cyd Disk Format

Human-readable disk images with a plain text SECTOR header and a raw data blob. Readable in any text editor.

cynext System

JSON files define shell commands with builtin:, lua:, or vfs: actions. Drop in ~/disk/cynext/ and boot.

oscyn API

Lua library that routes all OS requests through the kernel syscall interface. No raw os.execute calls.

Host Disk

~/disk/ is the live system root. Files written inside Cynessa appear instantly on the host filesystem.

CynIDE

Nano-style TUI editor with syntax highlighting for Lua, C and Python. ^R to compile and run in-place.
